- **Milestone ID:** ms-08-scheduler-platform
- **Roadmap ID:** feed-platform

- **Created at:** 2026-05-04T00:00:00Z
- **Last updated:** 2026-05-04T00:00:00Z

## 目的

Cron 的な定期処理基盤を構築し、入力プラグインの周期起動・実行監視・Pub/Sub 等によるバッチ処理分割を可能にする。AI 要約機能 (`ms-09`) の周期起動も同基盤で扱える土台を成立させる。

## 到達点 (定性)

- 定期実行基盤が成立し、`ms-06-input-plugin-platform` で実装された参照アダプタを定期周期で起動できる
- 実行監視機能 (実行成功 / 失敗 / 遅延 / リトライ状況の観測) が成立し、人間が運用状況を把握できる
- Pub/Sub 等の機構を用いて、大量の取得対象を複数バッチに分割実行できる (バッチ分割の動作が確認可能)
- 定期実行基盤と取得システムの結合方式 (L6) が確定し、ADR として記録されている (Cron 直接呼出 / IQ 経由トリガーメッセージ / その他のいずれを採るか)
- AI 要約の周期起動が同基盤で扱える設計が design.md 上で示されている (実装は `ms-09` で行う)
- 定期実行基盤のテストケース (周期起動 / 失敗時 retry / 監視ログ) が網羅され `vp test` で green を維持

## スコープ

- 対象モジュール: 採用ワークスペース上の定期実行基盤パッケージ、実行監視ログ収集、Pub/Sub バッチ分割機構
- 対象機能: Cron 的定期起動、入力プラグイン参照アダプタの周期実行統合、実行監視、バッチ分割
- 対象 ADR: L6 (定期実行基盤と取得システムの結合方式)、AI 要約周期起動を同基盤で扱う設計方針

## 非スコープ

- 入力プラグイン参照アダプタの実装 — `ms-06` の責務 (本マイルストーンでは ms-06 の参照アダプタを呼び出す側に立つ)
- AI 要約機能の実装 — `ms-09` の責務 (本マイルストーンでは AI 要約周期起動の設計余地を確保するに留める)
- 期間限定共有の自動失効ジョブ — `ms-04-auth-shared-access` で先行 Pull 型実装済、ms-08 完了後に Push 型ジョブとして補強する場合は ms-04 配下サイクルの追加 dev-workflow サイクルで対応
- 具体的サービス選定 (AWS EventBridge / Cloudflare Cron Triggers / Google Cloud Scheduler 等) は配下 `dev-workflow` サイクル Step 3 (Design) で確定し、Intent の「サーバレス原則」に整合する選定に留める

## 補足 / 留意事項

- L6 の確定により、入力プラグインのトリガー経路が「Cron → 関数直呼出」「Cron → IQ 投入」のいずれかに統一される。本マイルストーンの選択は `ms-06` で実装済みの参照アダプタの呼び出し形と整合させる
- 実行監視は最低限「成功 / 失敗 / 開始時刻 / 終了時刻 / リトライ回数」を観測可能とする。詳細メトリクス (p95 レイテンシ等) は将来別ロードマップ
- バッチ分割は「100 個の取得対象を 10 個ずつのバッチに分割し並列起動」できる程度の粒度を最低保証線とする
