# Milestone: Auth — 期間限定共有

- **Milestone ID:** ms-04-auth-shared-access
- **Roadmap ID:** feed-platform
- **Status:** planned
- **Created at:** 2026-05-04T00:00:00Z
- **Last updated:** 2026-05-04T00:00:00Z

## 目的

任意の対象 (フィード / 出力結果 / 集計ビュー等) を**任意期間 / 任意対象に対して共有**できる仕組みを構築する。Intent ユーザー要望「できればこれも RBAC で関してみたい」に応え、可能な限り既存 RBAC レイヤと統合する形で実装する。

## 到達点 (定性)

- ユーザーが任意のリソース (フィード / 出力 / 集計ビュー等) に対し、共有相手 (個別ユーザー / Organization / リンク受信者全員 等) と共有期間 (永続 / 期限付き) を指定して共有できる
- 共有を受けた相手 (認証済みユーザー / 未認証リンク受信者) が共有リソースを期間内のみ閲覧 / 操作できる
- 共有期間が満了すると自動的に共有が失効する (= リソースアクセスが拒否される)
- 共有の発行 / 失効 / 有効期間の確認が認可レイヤを通して観測可能
- 期間限定共有が RBAC レイヤと統合された設計または独立ポリシーレイヤとして共存する設計のいずれかに確定し、ADR として記録されている
- 共有許諾 / 拒否 / 期限切れのテストケースが網羅され `vp test` で green を維持

## スコープ

- 対象モジュール: 認証基盤パッケージの共有モデル / 共有失効ジョブ / 共有判定 API
- 対象機能: 共有リンク発行 (期間 / 対象指定)、共有 ACL 判定、期間満了による自動失効
- 対象 ADR: 共有 ACL の RBAC 統合方式 (Guest ロール + 期限拡張 / 専用ポリシーレイヤ) のいずれを採用するか

## 非スコープ

- 認証 / 認可基盤そのものの構築 — `ms-02` / `ms-03` の責務
- 共有対象となる業務リソース (フィード / 出力 / 集計) の実装 — `ms-05` 以降で実装される
- 動的ロール / ABAC / カスタムポリシー (Intent 非スコープ)
- マーケットプレイス / プラグイン化された共有機構 (Intent 非スコープ)

## 補足 / 留意事項

- Intent 未解決事項「期間限定共有の RBAC 統合方針」の最終決定は本マイルストーンの配下 `dev-workflow` サイクル Step 1〜3 で行う
- 共有判定がアクセス毎に評価される (Pull 型) ため、本マイルストーン完了時点で `ms-08-scheduler-platform` 未完了でも機能成立する設計を採る
- 共有対象となる業務リソース (フィード / 出力 / 集計) は本マイルストーン時点では未存在のため、`ms-05` 以降のリソースが共有対象として登録される際の API 形 (リソース ID + 共有指定) のみを抽象化して提供する
