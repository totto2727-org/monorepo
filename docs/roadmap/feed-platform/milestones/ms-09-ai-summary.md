# Milestone: AI Summary

- **Milestone ID:** ms-09-ai-summary
- **Roadmap ID:** feed-platform
- **Status:** planned
- **Created at:** 2026-05-04T00:00:00Z
- **Last updated:** 2026-05-04T00:00:00Z

## 目的

任意トリガー (期間 / 取得時条件等) で対象フィードを要約する AI 要約機能を構築する。永続化基盤からの読み出しと出力プラグイン経由での配信に依存する形で、End-to-End の要約生成 → 配信フローを成立させる。

## 到達点 (定性)

- ユーザーが指定したトリガー条件 (期間範囲 / 取得時条件等) に基づき、対象フィードが AI モデルで要約される
- 要約結果が Event Source of Truth に新規イベントとして記録され、Projection / Query API 経由で出力プラグイン (ms-07 の参照アダプタ) に配信される
- 定期実行基盤 (`ms-08`) と統合され、AI 要約の周期起動が動作する
- AI 要約のテストケース (トリガー判定 / 要約生成 / 配信) が網羅され `vp test` で green を維持 (AI モデル呼び出しはモック / 録画レスポンスで代替し、AI モデル自体の品質評価はスコープ外)
- AI モデル選定軸が ADR として記録されている (具体的事業者選定は本ロードマップでは大枠のみ)

## スコープ

- 対象モジュール: 採用ワークスペース上の AI 要約機能パッケージ、トリガー判定、要約生成、Event Source of Truth への要約イベント追記、配信統合
- 対象機能: トリガー条件 (期間 / 取得時条件) の評価、AI モデル呼び出し、要約イベント記録、定期実行基盤からの起動統合
- 対象 ADR: AI モデル選定軸、要約イベントのスキーマ、トリガー条件の表現方式

## 非スコープ

- 永続化基盤 (Event Source of Truth / Projection / CQRS) — `ms-05` の責務 (本マイルストーンでは「読み出し + 要約イベント追記」のクライアント側に立つ)
- 出力プラグイン基盤 / 参照アダプタ — `ms-07` の責務 (本マイルストーンでは要約結果を出力プラグインに渡す側に立つ)
- 定期実行基盤 — `ms-08` の責務 (本マイルストーンでは ms-08 の定期実行基盤を利用する側に立つ)
- AI モデル自体の品質評価 / プロンプトエンジニアリングの本格的最適化 (個人開発スコープでは「動作する要約が生成される」までを目標)
- 全文検索 / ベクトル検索 / セマンティック検索 (Intent 非スコープ)

## 依存マイルストーン

- `ms-05-persistence-event-store`: 永続化基盤からの読み出しと要約イベント追記が前提
- `ms-07-output-plugin-platform`: 要約結果を出力プラグイン経由で配信するため
- `ms-08-scheduler-platform`: AI 要約の周期起動を定期実行基盤に統合するため

AI 要約は既存基盤 (永続化 / 出力 / 定期実行) を組み合わせる結合機能のため、新規構築要素が少なく単一サイクルで完遂可能。AI モデル呼び出しの実装は外部 API クライアントレベルで完結する。

## 補足 / 留意事項

- AI モデル提供事業者 (OpenAI / Anthropic / Google / セルフホスト等) の選定は本ロードマップでは大枠のみ。配下 `dev-workflow` サイクル Step 1〜2 で「要約品質 / コスト / レイテンシ」の選定軸を確定し、具体的事業者選定は ADR 化する
- AI モデル呼び出しは独立サーバレス関数 (= マイクロサービス境界) として展開し、Event Sourcing 経路に統合する
- 要約結果は新規イベントとして Event Source of Truth に追記される (元フィードを書き換えない)
