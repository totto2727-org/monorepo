- **Status:** planned
- **Created at:** 2026-05-04T00:00:00Z
- **Last updated:** 2026-05-04T00:00:00Z

## 目的

`ms-05` で確定した**入力プラグイン契約スケルトン**に従い、独立サーバレス関数 (= マイクロサービス境界) として展開される入力プラグイン基盤と、最低 1 つの参照アダプタ (RSS 推奨) を成立させる。新たな入力源を**コードレベルで後付け可能**な構造を確立する。

## 到達点 (定性)

- 入力プラグイン契約 (interface) に従う参照アダプタが少なくとも 1 つ動作し、外部入力源からフィードを取得 → インプット Queue (IQ) に投入 → イベント記録 → Event Source of Truth に保存 まで end-to-end で通る
- 参照アダプタが**独立サーバレス関数**として実装され、他プラグインと独立にデプロイ・スケール可能な構造であることが design.md 上で示されている
- 「新たな入力アダプタを追加する手順」が文書化され、実際に追加コードのみで拡張できることが構造的に確認できる (動的ロード機構ではなくコードレベル契約による)
- 取得情報用 DB の必要性 (L4) について本マイルストーン内で確定し、ADR として記録されている (取得システムが状態を持つ / イベントストアに状態集約 / 軽量 KV のいずれを採るか)
- 入力プラグイン基盤のテストケース (契約準拠 / 参照アダプタ動作 / 失敗時 retry) が網羅され `vp test` で green を維持
- 認証認可基盤と統合され、認証済みユーザーの Organization スコープに紐付くフィードとして取り込まれる

## スコープ

- 対象モジュール: 採用ワークスペース上の入力プラグイン基盤パッケージ、参照アダプタ (RSS 推奨、ただし配下 oh-my-codingagent execution サイクル Step 1 で最終確定)
- 対象機能: 入力プラグイン契約準拠の参照アダプタ実装、IQ 投入、取得状態管理 (L4 の選択結果に応じた実装)
- 対象 ADR: L4 (取得情報用 DB の必要性) の確定、参照アダプタ選定理由

## 非スコープ

- 永続化基盤 (Event Source of Truth / Projection / CQRS) — `ms-05` の責務
- 入力プラグイン契約 (interface) のスケルトン定義 — `ms-05` の責務
- 出力プラグイン基盤 — `ms-07` の責務
- 定期実行基盤 (周期取得トリガー) — `ms-08` の責務 (本マイルストーンでは手動 / テストトリガーで動作確認、定期実行統合は ms-08)
- HTML 解析 / X リスト等の追加参照アダプタ実装 (本マイルストーンでは 1 アダプタを参照実装として完成させる、追加アダプタは将来の独立マイルストーンまたは本マイルストーン内 1:N サイクルで対応可能)

## 依存マイルストーン

- `ms-03-auth-rbac-organization`: 取得したフィードを Organization スコープに紐付けるため、認証認可基盤 (RBAC + Organization) が前提
- `ms-05-persistence-event-store`: 入力プラグイン契約スケルトンと Event Source of Truth が前提

## 想定 oh-my-codingagent execution サイクル数

1 (推奨)

`ms-05` で契約スケルトンが先行確定しているため、本マイルストーンは「契約準拠 + 1 つの参照アダプタ実装 + L4 確定」に集約され、単一サイクルで完遂可能。RSS / HTML 解析 / X リストを並行実装したい場合は 1:N (3 サイクル) を許容するが、最低限の参照実装は 1 アダプタで十分。

## 補足 / 留意事項

- 参照アダプタの第一候補は RSS (公開仕様あり / 認証不要 / 動作確認容易)。HTML 解析 / X リストは認証 / レート制限 / 仕様変動の懸念があるため後続検討
- 取得周期は本マイルストーン時点では手動トリガー / 単発実行で動作確認し、定期実行統合は `ms-08-scheduler-platform` 完了後に行う
- L4 の確定結果が「取得システムが状態を持つ」となった場合、サーバレス関数のステートレス性とのトレードオフを ADR 内で明記する
