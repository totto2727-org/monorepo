# TC-032: 並列配置 + retrospective 集約言及の目視検証 (SC-14)

- **対象成功基準:** SC-14 (Intent Spec — `docs/workflow/<identifier>/` と `docs/roadmap/<roadmap-id>/` の並列配置の図示/説明、および roadmap retrospective が `docs/retrospective/` 集約に保存される旨の明記)
- **TC ID:** TC-032
- **実行主体 / 検証スタイル:** `manual` × `inspection`
- **実行タイミング:** Step 8 Validation。`docs/workflow/2026-04-29-add-dev-roadmap-skill/validation-report.md` 作成時に Validator が実行する

## 目的

`dev-roadmap/SKILL.md` および `shared-artifacts/SKILL.md` の通読により、SC-14 が要求する以下 3 観点が確実に明記されているかを目視判定する。単純文字列検索 (`ggrep`) では「並列配置」の意図表現が文言依存で揺れるため、本テストは目視判定として TC-031 と独立に配置されている。

検証する 3 観点:

- **(a) 並列配置の図示または説明:** `docs/workflow/<identifier>/` と `docs/roadmap/<roadmap-id>/` が並列配置である旨の図示 (Mermaid) または説明文
- **(b) `docs/retrospective/` 集約への保存:** roadmap retrospective が `docs/retrospective/<集約名>` に保存される旨の明記
- **(c) `roadmap-` prefix 命名規則の明記:** roadmap retrospective のファイル名 prefix 命名規則 (`docs/retrospective/roadmap-<roadmap-id>.md`) の明記

## 前提条件

以下 2 ファイルが本サイクル完了時点でリポジトリに存在することを実行前に確認する。1 件でも欠落していれば本テストは実行不能のため Blocker として Main に報告する。

- `plugins/dev-workflow/skills/shared-artifacts/SKILL.md` (T8 成果物 = 既存ファイルへの追記)

確認コマンド例:

- `plugins/dev-workflow/skills/shared-artifacts/references/roadmap-retrospective.md`

## 手順

### 手順 (a): 検証対象 2 ファイルの通読 (上限 15 分)

1. 以下の順で 2 ファイルを通読する。読み返しは可。
   1. `plugins/dev-workflow/skills/dev-roadmap/SKILL.md`
   2. `plugins/dev-workflow/skills/shared-artifacts/SKILL.md`
2. 通読中に「保存構造」「ディレクトリ」「retrospective」「prefix」等のキーワードを含むセクションを重点的に観察する。

### 手順 (b): 観点 (a) — 並列配置の図示/説明の存在確認

以下のいずれかが**少なくとも 1 つ**存在することを確認する:

1. **Mermaid 図** (`graph LR` / `graph TD` / `flowchart` 等) で `docs/roadmap/` と `docs/workflow/` (もしくは具体パス `docs/roadmap/<roadmap-id>/` と `docs/workflow/<identifier>/`) が並んで描かれている。
2. **本文の説明文** で「並列配置」「並列に配置」「parallel layout」「並列ディレクトリ」等の語句で 2 ディレクトリの並列性が明示されている。
3. **見出しと隣接配置** で `docs/roadmap/` 構造図と `docs/workflow/` 構造図が同じセクション内・近接位置に配置され、文中に「並列」「並列配置」「同レベル」「同階層」「sibling」等の表現が含まれている。

該当箇所をファイルパスと行番号で記録する (例: `dev-roadmap/SKILL.md:355-396` / `shared-artifacts/SKILL.md:145-158`)。

### 手順 (c): 観点 (b) — `docs/retrospective/` 集約への保存記述の存在確認

以下を**少なくとも 1 つ**確認する:

1. `docs/retrospective/` というパス文字列が、roadmap retrospective の保存先として明示的に書かれている (例: 「`docs/retrospective/roadmap-<roadmap-id>.md` に保存される」)。
2. 「集約」「集約ディレクトリ」「集約形式」等の語句で、roadmap retrospective が `dev-workflow` retrospective と同じディレクトリに集約される旨が明記されている。

該当箇所をファイルパスと行番号で記録する。

### 手順 (d): 観点 (c) — `roadmap-` prefix 命名規則の明記

以下を**少なくとも 1 つ**確認する:

1. `docs/retrospective/roadmap-<roadmap-id>.md` 形式のパスが直接記述されている。
2. 「`roadmap-` prefix」「`roadmap-` プレフィックス」等の語句で、命名規則として明記されている。
3. 表 / 箇条書きで `dev-workflow` retrospective (`<identifier>.md`) と `dev-roadmap` retrospective (`roadmap-<roadmap-id>.md`) の対比が示されている。

該当箇所をファイルパスと行番号で記録する。

### 手順 (e): チェックリスト記入

以下のチェックリストに YES / NO を記入する。

| 観点                                           | 検出ファイル / 行番号 | YES / NO |
| ---------------------------------------------- | --------------------- | -------- |
| (a) 並列配置の図示/説明                        |                       |          |
| (b) `docs/retrospective/<集約名>` への保存記述 |                       |          |
| (c) `roadmap-` prefix 命名規則の明記           |                       |          |

## PASS 判定基準

**(a) (b) (c) の 3 観点すべてが YES** のときのみ PASS と判定する。

- 1 つでも NO がある場合は FAIL。
- 観点ごとに「`dev-roadmap/SKILL.md` または `shared-artifacts/SKILL.md` のいずれか 1 ファイル以上で確認できれば YES」とする (両ファイルに記述があれば望ましいが必須ではない)。
- ただし観点 (c) について、`dev-roadmap/SKILL.md` または `shared-artifacts/SKILL.md` 以外のファイル (例: `references/roadmap-retrospective.md`) のみに記述されている場合は本 TC では NO と判定する (TC-033 の対象範囲)。

## FAIL 判定例

以下のいずれかに該当する場合は FAIL とする。FAIL 時はその根拠と発見事項を `validation-report.md` に記録し、対象ファイル (T1 / T8 成果物) の改善要否を Main に報告する。

- **観点 (a) 不在:** 2 ファイルのいずれにも、`docs/roadmap/` と `docs/workflow/` の並列配置を示す図 / 説明文 / 隣接構造が見当たらない。
- **観点 (b) 不在:** `docs/retrospective/` パスへの保存記述または「集約」を示す表現が 2 ファイルのいずれにも見当たらない。
- **観点 (c) 不在:** `roadmap-` prefix の具体形 (`roadmap-<roadmap-id>.md`) または「prefix 命名規則」の語句が 2 ファイルのいずれにも見当たらない。
- **誤った場所への記述:** 観点 (a) (b) (c) のいずれかが、`dev-roadmap/SKILL.md` / `shared-artifacts/SKILL.md` 以外のファイル (例: `references/roadmap.md` のみ) にしか記述されていない。
- **記述が誤解を招く形:** 例えば「`docs/retrospective/` に保存される」とあるが prefix 命名が `dev-workflow` retrospective と同じ `<identifier>.md` 形式であるかのように読める場合は、観点 (c) を NO とする。
