## Detailed sections

### m-1 detail: `generate-docs-components-build.ts` sequential fetch loop

Source location: `plugins/totto2727-coding/.script/generate-docs-components-build.ts:88-107`.

```ts
for (const slug of sortedSlugs) {
  const llmsUrl = `${BASE_URL}/llms.mdx/${slug}`
  try {
    const response = await fetch(llmsUrl)
    // ...
  } catch (error) {
    console.warn(`Skipping ${slug}: ${error}`)
  }
}
```

- **Observation:** with N slugs, total wall time ≈ N × (RTT + server processing). Each iteration awaits the previous fetch.
- **Why acceptable now:** Intent Spec `intent-spec.md:163-165` excludes "規約の中身" rewrites; this cycle migrates the script verbatim aside from the rename + null guard + frontmatter template fixes. Touching the loop semantics would expand scope.
- **Cycle introduced no regression:** the pre-existing version `plugins/components-build/.script/generate-skill.ts` had the same loop. Diff between old and new (per commit 347d5c6) is limited to: filename, output dir constant, frontmatter `name:` template string, comment update (L131-132 reference), and the null guard at L12-17.
- **Recommended follow-up (out of scope this cycle):** wrap the per-slug fetch in `Promise.all` with `Promise.allSettled` semantics and a small concurrency cap (e.g. 8) to keep server politeness. Estimated speed-up: ~5-8× depending on server.

### m-3 detail: `docs-moonbit/SKILL.md` line count

`docs-moonbit/SKILL.md` is 313 lines, just over the 300 hard-cap that applies to `coding/SKILL.md` and `test/SKILL.md`. SC-3 (Intent Spec L249) explicitly bounds only the latter two files; the cap was deliberately not applied to the auto-generated docs skills because their body is dominated by an auto-generated link table.

- **Composition** of `docs-moonbit/SKILL.md`: frontmatter (~7 lines) + license headers (~6 lines) + Related Skills (~3 lines) + Introduction body (~262 lines copied from upstream) + Related Documentation table (~33 lines linking 26 reference files).
- **Activation-time impact:** Claude Code reads the frontmatter `description:` for activation classification (this is short — 3 sentences). The body is loaded on-demand, so the 313 lines do not enter the per-session activation budget; they are paid only when the user invokes the skill.
- **Why "Minor":** the file is long by absolute measure but its weight is at the on-demand layer, not at activation. No regression vs. the source `plugins/moonbit/skills/moonbit-docs/SKILL.md` (the structure was preserved verbatim per user instruction "現在のスキル構造を維持").
