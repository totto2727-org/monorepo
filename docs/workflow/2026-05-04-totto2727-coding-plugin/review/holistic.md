**Recommended fix (single commit, ~5 minutes):** replace `moonbit-bestpractice` with `mbt-bestpractice` (the new in-plugin reference name) and `effect-*` with `coding` (matching A4's reduce-to-example-of-1 strategy). Sentences should keep the "e.g." framing.

### M-2 detail: stale `process-moonbit-docs.ts` references inside the migrated script body

`plugins/totto2727-coding/.script/generate-docs-moonbit.ts` lines 1-18 (header comment + Usage error):

```ts
// Fetches MoonBit official documentation markdown and generates skill reference files.
//
// Source pages:
//   - Language: https://docs.moonbitlang.com/en/latest/language/index.html
//   - Toolchain: https://docs.moonbitlang.com/en/latest/toolchain/index.html
//
// Download links (pass these as arguments):
//   sfw deno run --allow-net --allow-read --allow-write .script/process-moonbit-docs.ts \    ← STALE
//     https://docs.moonbitlang.com/en/latest/_downloads/78d8998d78e60fd7b1b2f4c9bb2819fb/summary.md \
//     https://docs.moonbitlang.com/en/latest/_downloads/7e67a7065021137fc80a2750bac9ee32/summary.md

import { join } from 'jsr:@std/path@1.1.4'

const urls = Deno.args
if (urls.length === 0) {
  console.error('Usage: deno run --allow-net --allow-read --allow-write .script/process-moonbit-docs.ts <URL> [URL...]')   ← STALE
  Deno.exit(1)
}
```

Two occurrences of `process-moonbit-docs.ts` should be `generate-docs-moonbit.ts`. T5 / `task-plan.md:91-105` enumerated 4 edit categories (null-guard, output dir, skillContent template, relatedSkillsBlock template) but did not call out the header comment + Usage error message, so the implementer missed them. The companion `generate-docs-components-build.ts` does not have this issue (its header at line 5 already says `generate-docs-components-build.ts`).

**Severity rationale (Major, not Blocker):** SC-7 (`deno check`) passes; functional behavior is correct. The bug surfaces only when a user runs the script with zero args — they get a misleading error message pointing to a non-existent file. UX bug, not a functional regression.

**Recommended fix:** 2 line edit.

```ts
//   sfw deno run --allow-net --allow-read --allow-write .script/generate-docs-moonbit.ts \
console.error('Usage: deno run --allow-net --allow-read --allow-write .script/generate-docs-moonbit.ts <URL> [URL...]')
```

**Round 2 outcome (resolved by 70e19f7, T5-r2):** both occurrences updated in commit 70e19f7. The header comment line was a 1:1 substitution; the `console.error(…)` line was also reformatted to a multi-line statement (4 inserts / 2 deletes total in the diff, but logically a 2-string substitution). Re-grep of `process-moonbit-docs` over the file returns 0 hits → status flipped to `fixed`. SC-7 (`deno check`) remains exit 0.
