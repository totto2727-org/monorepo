# atlas-kysely-gen

Atlasの`schema.hcl`からKyselyの型定義を生成するCLIツールです。

- DB接続不要、Atlas CLI不要
- `ariga.io/atlas`のSQLite HCLコーデックを直接使用するため、型評価が正確
- `kysely-codegen`互換の出力形式

## インストール

```bash
go install github.com/your-org/atlas-kysely-gen@latest
```

またはソースから:

```bash
git clone https://github.com/your-org/atlas-kysely-gen
cd atlas-kysely-gen
go build -o atlas-kysely-gen .
```

## 使い方

```bash
# 標準出力に出力（識別子はそのまま）
atlas-kysely-gen --input schema.hcl

# ファイルに出力
atlas-kysely-gen --input schema.hcl --output src/db/types.ts

# 短縮形
atlas-kysely-gen -i schema.hcl -o src/db/types.ts

# KyselyのCamelCasePluginでカラム名とテーブル名を変換
atlas-kysely-gen -i schema.hcl --camel-case
```

### フラグ

| フラグ               | デフォルト | 説明                                                                                                                                                                                                                     |
| -------------------- | ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `--input` / `-i`     | (必須)     | 入力schema.hclへのパス                                                                                                                                                                                                   |
| `--output` / `-o`    | 標準出力   | 出力`.ts`ファイルへのパス                                                                                                                                                                                                |
| `--camel-case`       | `false`    | `true`の場合、Kyselyの公式`CamelCasePlugin`を使用してカラムとテーブルのDBキーを変換します（デフォルトオプション = `upperCase=false`）。`false`の場合は恒等変換（変換なし）を使用します。動作はkysely-codegenの`--camel-case`フラグと1:1で一致します。 |

## TSプロジェクトへの統合

`package.json`のスクリプトに追加する場合:

```json
{
  "scripts": {
    "db:codegen": "atlas-kysely-gen -i schema.hcl -o src/db/types.ts"
  }
}
```

または、Goのみインストール済みで`go run`を使用する場合:

```json
{
  "scripts": {
    "db:codegen": "go run github.com/your-org/atlas-kysely-gen@latest -i schema.hcl -o src/db/types.ts"
  }
}
```

## 出力例

入力 `schema.hcl`:

```hcl
table "users" {
  schema = schema.main
  column "id" {
    type = int
    null = false
  }
  column "email" {
    type = text
    null = true
  }
  column "status" {
    type = enum("active", "inactive")
    null = false
  }
}
```

出力 `types.ts`:

```typescript
/**
 * This file was generated by kysely-codegen.
 * Please do not edit it manually.
 */

export interface Users {
  email: string | null
  id: number
  status: 'active' | 'inactive'
}

export interface DB {
  users: Users
}
```

## 型マッピング

| Atlas / SQLite型                    | TypeScript型     |
| ----------------------------------- | ---------------- |
| `int`, `integer`, `*INT*`           | `number`         |
| `text`, `*CHAR*`, `*CLOB*`          | `string`         |
| `real`, `*FLOAT*`, `*DOUBLE*`       | `number`         |
| `numeric`, `decimal`                | `number`         |
| `boolean`, `bool`                   | `boolean`        |
| `blob`                              | `Uint8Array`     |
| `date`, `datetime`, `timestamp`     | `string`         |
| `json`, `jsonb`                     | `string`         |
| `enum("a","b")`                     | `"a" \| "b"`     |
| `null = true`                       | `T \| null`      |

## 開発

```bash
go run *.go -i fixture/schema.hcl
```

## ロードマップ

- [x] **`--camel-case`オプション** — kysely-codegenと同様の単一ブール値フラグ。Kyselyの`CamelCasePlugin`（デフォルトオプション）を使用して識別子をキャメルケースに変換するか、変換しない（恒等変換）かを切り替えます。
- [x] **kysely-codegen互換のケース変換ロジック** — JSオリジナルから`toKyselyCamelCase` / `toKyselyPascalCase`を忠実に移植。`upperCase=false`の場合、SCREAMING_SNAKE_CASE（例：`USER_ID`）はkysely-codegenと同様に意図的にそのまま維持されます。
- [x] **`Generated<>`ラッパーサポート** — デフォルト値または`auto_increment`を持つカラムを`Generated<T>`でラップします。また、`import type { Generated } from "kysely"`を自動出力します。
- [ ] **複数データベース方言のサポート** — 現在はSQLite HCLのみ対応。PostgreSQL、MySQLなどを追加予定。
- [ ] **ウォッチモード** — 入力HCLファイルの変更を検出し、型定義を自動的に再生成します。
