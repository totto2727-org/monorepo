# 2. `geo` algorithms ↔ `src/geo/2d/`

Port package: `totto2727/geo-mbt/geo/2d` (`src/geo/2d/pkg.generated.mbti`).

Rust uses extension traits dispatched via blanket impls (e.g. `Area for
Polygon<T>`). The port mirrors that surface with **MoonBit traits as the
canonical entry point**: every per-type behaviour that has more than one
implementing geometry is fronted by a trait that lives in the same
algorithm-layer source file as its private `<op>_<shape>` helpers
(e.g. `HasArea` in `area.mbt`, `Bounded` in `bounding_rect.mbt`,
`LinesCarrier` in `lines_iter.mbt`, `MapCoords` in `map_coords.mbt`).
Type-shape traits (`IsEmpty`, `IsClosed`, `HasLength`) live in
`src/geo/2d/type/traits.mbt` since they wrap inherent methods on each
struct. The historical `<op>_<shape>` free functions still exist but
are **package-private** — they back the trait impls so whitebox
`*_wbtest.mbt` files can target them, but external callers must dispatch
through the trait.

#### Type-shape traits — `src/geo/2d/type/traits.mbt`

These wrap the inherent methods that already live on each `type/<struct>.mbt`,
so dot-syntax `obj.is_empty()` continues to work for direct callers while
trait dispatch (`IsEmpty::is_empty(obj)`) is available for generic code.

| Port trait  | Members                   | Implementing types                                                                                                                                                       | Mirrors upstream          |
| ----------- | ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------- |
| `IsEmpty`   | `is_empty(Self) -> Bool`  | LineString, MultiPoint, MultiLineString, MultiPolygon, GeometryCollection, Polygon, Geometry (7)                                                                         | `HasDimensions::is_empty` |
| `IsClosed`  | `is_closed(Self) -> Bool` | LineString, MultiLineString (2)                                                                                                                                          | `LineString::is_closed`   |
| `HasLength` | `length(Self) -> Int`     | LineString, MultiPoint, MultiLineString, MultiPolygon, GeometryCollection (5; collection cardinality, **NOT** physical length — use `HasEuclideanLength` for the latter) | `Vec::len` / `Slice::len` |

#### Default-value traits

| Port trait | Implementing types                                                                                                                                                              | Notes                                                                                                |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| `Default`  | Coord, Point (via `derive(Default)` on `Double` fields); LineString, MultiPoint, MultiLineString, MultiPolygon, GeometryCollection (via `derive(Default)` on `Array[T]` fields) | Returns the canonical empty / zero value. Equivalent to `Coord::zero` / `<type>::empty()` factories. |

Tests: trait coverage is exercised inside each algorithm's per-source
`*_test.mbt.md` doctest (post-`3a4914d` migration: every `<algo>` test
file dispatches the trait method on every implementing type). The
previous cross-cutting `traits_test.mbt` file was retired when the
doctest migration absorbed its sweeps.

### 2.3 Property tests (no direct upstream counterpart)

`property_test.mbt` — 11 invariant-style cases. They exercise composed
behaviour (e.g. _signed_area flips sign on reversal_, _bounding rect
contains every coord_, _translate is invertible_, _simplify preserves
endpoints_, _line_locate ∘ line_interpolate ≈ id_, _orient_polygon is
idempotent_, _hausdorff_distance is symmetric_, _scale preserves shape_).
