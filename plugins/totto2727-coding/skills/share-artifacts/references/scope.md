# Shared Artifact Scope

> Document type: concrete skill-scope policy.

Use this skill to create or review a roadmap, milestone, roadmap retrospective, or ADR with the matching guide and template.

This skill does not define implementation workflows, role systems, commit protocols, `progress.yaml` mutations, or unrelated project documentation. Use the owning workflow or CLI for those operations.

Examples that belong here include writing `roadmap.md`, copying the milestone template, or reviewing an ADR against its quality criteria. Starting implementation, editing CLI-owned state by hand, and updating an `AGENTS.md` file do not belong here.
